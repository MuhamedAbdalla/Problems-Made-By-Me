Bedo hates long statements, so, he will go to the problem's explanation directly.

You have a tree of n nodes and you have qq query of two types.

First one is (tt = 1), aa, bb which means to update each node in subtree of node aa by value b.

Second one is (tt = 2), aa, bb which means that you should give us the product of the weights on the nodes in subtree of aa and bb.

for more explaination, define F(x) as the product of weights on the nodes in subtree of node x, then the answer for node a, b is F(a) * F(b).

If answer is greater than or equal 10181018, then print −1−1.

Input

First line number of nodes n. (1≤n≤3∗105)(1≤n≤3∗105).

the next n - 1 line contains u, v which means there is an edge between u - v. (1≤u,v≤3∗105)(1≤u,v≤3∗105).

Next line is qq. (1≤q≤105)(1≤q≤105).

The next qq lines will be tt, aa, bb. (1≤t≤2)(1≤t≤2), (1≤a,b≤n)(1≤a,b≤n).

Output

if answer greater than 10181018, then print -1.

otherwise print the answer.

Example

Input
13
1 2
2 3
2 4
2 5
4 6
4 7
1 8
8 9
8 10
9 13
10 11
10 12
3
1 2 2
1 4 3
2 2 4
Output
5832

Note

All inputs are integers.

All nodes have weight zero initially.
